Design system

What is it?

A design system is a systematic approach to product design/development

There are various definitions and interpretations of what exactly a design system is. Since they come in various styles, contexts and depths I'm going to use my own definition. A design system is a systematic approach to product design/development. Design systems are commonly used to expedite tasks, communicate between teams, and track the growth of a product. A design system should not misconstrued as a document with rigid guidelines. A design system is a collection of components and standards designed for efficiency and reusability.

I created Design System One because:

  • • I wanted to practice using a more systematic approach toward designing components
  • • I wanted to have ready-to-use components for future projects
  • • I wanted to delve into a project that I could iterate and improve upon

Research

Design systems

In order to increase my base knowledge on what a design system is, I decided to look through various design systems and frameworks available for viewing on the web. Included were Google's Material Design, NASA Graphics Standards Manual, and the Atomic Design methodology.

Scaling down

One UI Design Guidelines

Due to my familiarity with Galaxy phones and the layout of the system, I decided to begin my design system based off of Samsung's One UI Design Guidelines. Given the constraints of the project (1 contributor, time, etc.) I decided to begin the system with the components that make up the 'Settings' screens.

Aside from the high-level challenges such as identifying the grid system, screen dimensions, font-sizes, etc., It was important that I reference guidelines and take inventory of the components involved in Settings.

Components

Settings components

App (Settings)

App components exist in the Settings application. These components are made up of app bars, list items, lists, icons, images, etc.

Home

Home components exist on the phone's homescreen. This class of components mainly includes labeled app icons

System

System components are elements of the device's operating system. These components include system navigation bars, bottom bars, system icons, and dialogue menus.

Components I used to design the screens I've created so far. Most of these components contain smaller components within themselves (think Atomic Design) and are created with overrides and variants in mind.

Templates and screens

Templates and screen content

According to Atomic Design, templates are where organisms and components come together to create a cohesive design. In my design system the templates are components grouped together to represent the full height of the final design without the constraints of the screen dimensions.

Final screens

Final Settings screens

Templates are placed inside 480x1067 frames. There are a total of 12 frames - representing 6 different Settings screens - each with extended navigation and scrolled navigation states.

Final thoughts

Lessons learned

Building a design system takes thoughtful planning when designing components and organizing the heirarchy of components' naming conventions. Many components were inspired by Samsung's One UI Design Guidelines, but some creative liberties were taken with element padding, buttons, icons, and more.

Key takeaways

  • • Thoughtfully consider naming conventions before diving into the component design
  • • There's nothing wrong with following previous designs and industry standards when working on a creative project
  • • When designing components with overrides, don't neglect element constraints and alignments
